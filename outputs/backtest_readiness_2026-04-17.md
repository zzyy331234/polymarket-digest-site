## 一句话结论

**现在可以做“早期原型回测 / 数据管道验证”，但还不适合做正式 MRAS 全量回测定稿。**

原因很明确：
- **优点**：15m 级别原始价格数据已经积累到 **96.7 万条**，大部分高流动性市场已有 **24h~72h 连续数据**，足够做短窗趋势/均值回归的初测。
- **硬伤**：
  1. **7d 窗口仍然不够**（当前总覆盖仅 **116.28 小时 ≈ 4.85 天**）
  2. **`ta_snapshots = 0`**，说明 TA 快照链路实际上还没产出
  3. 期间存在几次**明显断档**，会污染 15m 连续 bar 和 rolling 特征

所以当前更适合：
- 验证数据质量
- 做 **4h / 24h 特征驱动** 的简化回测
- 做 carry_no / trend fallback 的初步评估

不适合：
- 用这批数据直接给 **MRAS 正式定稿**
- 使用依赖 **7d / RSI / MACD / TA snapshots** 的完整特征集做严肃结论

### 解读

这意味着：
- **15m bar 能构造**，不是不能用
- 但 bar 的“满格程度”并不一致
- 如果回测要求高稳定性，建议：
  - 只保留 `samples_per_bar >= 10` 的 bar
  - 或对稀疏 bar 打低质量分 / 直接过滤

**适合做中低精度回测，不适合假设每个 15m bar 都是满采样高质量 bar。**

## 6. 当前最现实的回测方式

### A. 现在就能做：简化版 MRAS 回测

建议先跑一个 **MRAS-Lite**：

**保留：**
- liquidity filter（>=20k / >=50k）
- carry_no
- trend（用 4h + 24h + slope + liquidity）
- mean_revert（只用近窗口 percentile，不标称 7d）

**暂时禁用 / 降权：**
- `chg_7d`
- 正式 `pct_rank_7d`
- `ta_snapshots` 相关 RSI/MACD 依赖项
- 任何假设“完整 7d 样本”的触发条件

### B. 1~3 天后可做：准正式短窗回测

等总覆盖过 **7 天** 后：
- `chg_7d` 开始可用
- `pct_rank_7d` 才算真正进入有效区间
- 趋势 / 均值回归判断会更稳

### C. 7~14 天后可做：更像样的策略审计

等你积累到 **2 周左右**：
- regime 稳定性更好
- 可按事件类型 / 流动性 / 到期时长做分层评估
- 才更适合讨论“是否允许小额实盘”这类问题

## 7. 我对“现在能不能回测”的判断

### 可以做的

1. **数据管道回测**
   - 检查 feature 生成是否稳定
   - 检查信号是否过少 / 过多
   - 检查高流动性市场是否持续产出可解释信号

2. **短窗收益验证**
   - 比如持有 4h / 12h / 24h 后的方向正确率
   - 做 trend / carry_no 的初步 precision 测试

3. **事件前过滤规则验证**
   - hours_to_event <= 24 / 48 的过滤逻辑可先试

### 不建议现在做的

1. 把这批结果当正式胜率结论
2. 用它决定放大实盘仓位
3. 用 7d 条件给 MRAS 定型
4. 把没有缺口处理的原始序列直接用于性能评估

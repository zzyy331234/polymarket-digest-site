# Polymarket 交易纪律 v2

> 目标：把“保命规则 / 研究规则 / 上线规则”拆开，避免把所有规则混成一层。
> 
> 核心原则：**长期稳定盈利增长 > 短期冲动收益**。
> 
> 当前阶段结论：系统已形成 paper 自动闭环，但策略仍处于 **paper-only**，未达到放开真钱自动化的条件。

---

## 0. 三层结构

### A. 保命层（Hard Rules）
这层规则是为了 **不爆、不乱、不上头**，默认不可放松。

### B. 研究层（Paper / Research Rules）
这层规则是为了 **加快学习、积累样本、持续修正策略**。

### C. 上线层（Release Gates）
这层规则决定 **能不能从 paper 进入 micro live，再进入小额自动化**。

---

## 1. 保命层（Hard Rules）

以下规则适用于所有阶段：paper、micro live、small auto、future live。

### 1.1 资金与仓位
- 禁止全仓、禁止单市场重仓。
- 单笔仓位必须受上限约束，当前默认：`max_single_size_usd = 3.0`。
- 总暴露必须受上限约束，当前默认：`max_total_exposure_usd = 6.0`。
- 单一 cluster 暴露必须受限，避免“看似分散，实则同一主题同一风险”。
- 任何时候都不能为了“补亏损”而加码。

### 1.2 交易频率
- 禁止连续乱下单。
- 同一策略窗口内，不允许因为情绪反复追单。
- 保留日内交易次数上限。
- 当日预算达到上限后，停止新增仓位。

### 1.3 熔断与刹车
- 连续亏损达到阈值，必须自动熔断。
- 日内损失达到阈值，必须停止开新仓。
- 任一异常触发时，`emergency_stop = true` 必须能立即阻断自动开仓。
- `live_trade_enabled = false` 时，一律不得进入真钱执行。

### 1.4 市场质量
- 流动性不足、不易成交、临近结算、价格几乎不动的市场，宁可错过，不要硬做。
- `blocking_flags` 命中时，不开仓。
- 低可交易性（low tradability）不是“保守过头”，而是防止死票和滑点陷阱。

### 1.5 禁止行为
- 禁止因为“上一笔亏了”而放大下一笔。
- 禁止因为“这个看起来稳”而单押。
- 禁止把 advisory warning 当成看不见。
- 禁止在策略未验证前，绕过 paper 直接上自动真钱。

## 2. 研究层（Paper / Research Rules）

这层的目标不是赚钱，而是：
- 验证哪些 regime 值得保留
- 找出哪些市场是“根本不动”的死票
- 优化开仓/平仓/过滤规则

### 2.1 信号分池
不再把“只做高置信度”写成全阶段一刀切，而是拆成三池：

#### 主交易池（paper 主池）
- 用于模拟真实策略表现
- 建议阈值：`confidence >= 0.58 ~ 0.60`
- 进入主统计、主复盘、主纪律约束

#### 高确信池（paper 精选池）
- 用于观察“如果更严会不会更稳”
- 建议阈值：`confidence >= 0.65`
- 单独统计命中率、收益率、flat rate

#### 实验池（research bucket）
- 用于探索中等置信度或新过滤器
- 建议阈值：`0.50 <= confidence < 0.58`
- 可以观察，但**不得混入实盘放行统计**

### 2.2 Regime 管理
- 每个 regime 独立复盘，不允许“总体看起来还行”掩盖某个 regime 持续失效。
- 如果某个 regime 出现以下任一情况，应降级或暂停：
  - flat rate 长期过高
  - 胜率明显低于其他 regime
  - 平均盈亏比明显差
  - 需要依靠少数偶发大单才能拉平整体结果

### 2.3 Cluster 管理
- cluster 必须单独做统计。
- 某些 cluster 即使表面信号很多，只要长期不动、流动性差、成交差，就应降权或屏蔽。
- 不再依赖“题材印象”拍脑袋，而是看：
  - flat rate
  - 成交质量
  - 振幅是否足够覆盖手续费/滑点/退出成本

### 2.4 平仓规则
paper 阶段允许继续试验，但必须记录版本：
- thesis invalidated
- time stop（如 24h / 48h）
- low tradability exit
- stale carry_no exit
- 事件临近退出

要求：
- 每次平仓必须写明 reason
- 每个 reason 的胜率和收益要单独统计
- 如果某个平仓规则系统性“砍掉潜在盈利”，就要重估

### 2.5 当前建议
基于近期结果，当前更建议：
- 保留 mean_revert 主线继续观察
- trend 不要放大，先继续 paper 审核
- carry_no 保持严格限制，必要时继续降级
- 对“几乎不动”的市场继续强化 low_tradability 过滤

## 3. 上线层（Release Gates）

不再采用“达标前 0 / 达标后 100”的二元切换。
改为：**分阶段放行**。

### Phase 0：Analysis Only
适用情形：
- 还在改规则
- 数据样本不足
- collector / scanner / review pipeline 不稳定

允许：
- 扫描
- 打分
- 生成候选
- 生成报告

禁止：
- 自动开仓
- 真钱执行

### Phase 1：Paper Trade
适用情形：
- 闭环已通，但策略仍在检验

要求：
- 自动 paper 闭环稳定运行
- trade log 完整
- 平仓原因完整
- 可以稳定输出 summary/report

当前系统属于这一阶段。

### Phase 2：Micro Live（人工确认）
进入条件建议：
- 连续足够多的 paper 样本（建议先累计 30~50 笔已关闭交易）
- 主池胜率、盈亏比、最大连续亏损达到最低标准
- 不再明显依赖单一 cluster 或单一偶发信号
- 系统连续运行稳定，无明显状态漂移/日志断裂

执行方式：
- 单笔极小额（如 1~2 USDC）
- 每天最多 1~2 笔
- 仅允许最稳的 1 个 regime
- 必须人工确认
- 保留一键全停

### Phase 3：Small Auto
进入条件建议：
- micro live 已经证明 paper 与真实成交偏差可接受
- 滑点、成交、退出表现稳定
- 连续一段时间没有出现纪律性事故

执行方式：
- 小额自动
- 更严格的 daily limit / exposure limit
- 继续人工巡检
- 任一异常立即回退到 Phase 2

### Phase 4：Scale Carefully
进入条件建议：
- 多阶段稳定
- 多 regime / 多 cluster 都验证过
- 有明确“为什么扩大规模仍然成立”的证据

原则：
- 逐步加，不跳级
- 每次只放大一点
- 放大后继续观察，不可自动乐观外推

## 4. 放行指标（建议版）

下面不是“今天立刻必须满足”的绝对真理，而是现阶段比较合理的门槛。

### 4.1 从 Paper → Micro Live
建议至少同时满足：
- 已关闭样本数：`>= 30`
- 胜率：`>= 55%`
- 平均盈亏比：`>= 1.3`（更理想是 `>= 1.5`）
- 最大连续亏损：`<= 3`
- flat rate：不能高到说明大部分票都不动
- 不能只靠单一 regime/cluster 撑结果

### 4.2 从 Micro Live → Small Auto
建议至少同时满足：
- 真实成交结果与 paper 偏差在可接受范围内
- 小额真钱连续一段时间无纪律事故
- 手续费/滑点/退出成本没有吞掉 edge
- 人工确认阶段没有发现“paper 看着行，实盘完全不行”的结构性偏差

---

## 5. v2 的一句话定义

> **保命规则更硬，研究规则更活，上线规则分阶段。**

也就是：
- 不放松生存纪律
- 不用一刀切扼杀研究样本
- 不把“是否实战”做成开关，而做成台阶

---

## 6. 当前执行建议（2026-04-19）

### 立即保留不动
- 稳定盈利增长 > 一切
- 禁止全仓
- 禁止连续乱下单
- 连续亏损熔断
- 日损熔断
- live 默认关闭

### 立即调整表述
把：
- “只做高置信度信号”

改成：
- **实盘只做高置信度；paper 允许主池 + 实验池双轨观察，但实验池不得进入放行统计。**

把：
- “策略达标才允许实战”

改成：
- **策略分阶段达标，才允许进入下一层级实战。**

### 当前阶段定位
- 当前仍属于 **Phase 1：Paper Trade**
- 暂不建议进入自动真钱
- 如要试真钱，也只能是 **Micro Live + 人工确认 + 极小额**

## 7. 后续落地动作

1. 在报告里把信号分成：高确信池 / 主池 / 实验池
2. 在 paper summary 里增加：
   - flat rate
   - regime 胜率
   - cluster 胜率
   - 平仓规则收益对比
3. 先累计足够关闭样本，再评估是否进入 micro live
4. 任何阶段出现纪律性事故，立即回退一层
